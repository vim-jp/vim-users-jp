問題

0

このコマンドが呼び出すrubyとして期待されるものは、0であったり、0であったり、はたまた0のように独自で用意したものであったりします。

0などのシェルでは、0などの設定ファイルに

0

などと記述することによって、単に0などとしてコマンドを呼び出したときに実際にどの場所のコマンドが呼び出されるか制御でき、またそのシェルから起動したVimもそれに準拠します。これはCUIでもGUIでも同様です。

しかしながら、シェルを経由せずVimを起動した場合。0などに記述したロードパスに関する情報は得られません。MacのDockなどから起動しつつ、そのVimであなたのホームディレクトリ以下にある0などを呼び出すにはどうすればいいのでしょうか。

解決法

vimprocを用います。vimprocはVimの変数0を参照してコマンドを実行します。

vimprocを用いてのコマンドの実行とVimのデフォルトの機能との対応は以下のようになります。

  • コマンドを単に実行する

    • Vim: 0
    • vimproc: 0
    • vimproc’: 0

  • コマンドを実行した結果をバッファに書き足す

    • Vim: 0
    • vimproc: 0
    • vimproc’: 0

  • コマンドを実行した結果を変数に格納する

    • Vim: 0
    • vimproc: 0

  • コマンドをバックグラウンドで実行する

    • Vim: 0
    • vimproc: 0

冗長で不便なので、キーマッピングを追加するか、あるいは0の使用をさけてかわりにvimshellを用いるとよいでしょう。

0

さて、0や0を間接的にもっとも頻繁に使うのはなんといってもquickrunでしょう。quickrunでvimprocを用いるためには、以下の設定を0に記述します。

0

余談ですが、著者はこれにquickrun分割場所指定もあわせ、以下のように指定しています。

0

これで、quickrunで好きなrubyを使いたい放題です。

補足

本Hackは一般的な外部コマンドの実行について述べましたが、この方法は特に、Rubyのような複数の同名のコマンドをインストールする必要があるようなコマンドで役に立ちます。著者はRuby 1.8.6, 1.8.7, 1.9.2, 1.9.3をすべて使用しているため、本設定が大変効いてきています。
